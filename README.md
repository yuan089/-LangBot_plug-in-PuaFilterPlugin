# 📘 PuaFilterPlugin：AI 人格修正與訊息安檢系統

**作者**：Aisin_Lychee  
**版本**：v4.0-Stable  

本插件是專為大語言模型（LLM）在即時通訊環境下設計的「動態安檢站」，位於模型輸出與使用者接收之間的最末端。

---

## ✨ 核心功能詳解

### 1. 🚫 言論重複懲罰 (Anti-Repetition)
- **功能描述**：自動識別並終止 AI 的「死循環」行為。  
- **何時有用**：當本地 7B/8B 模型因上下文過長或參數不穩，開始重複「我是AI，我是AI..」這類車軲轆話時，本功能會強制去重，只保留一句，並重新組合文案。

### 2. 🧠 人格重組與修正 (Personality Restoration)
- **功能描述**：防止「人格稀釋」。當 AI 輸出被代碼污染或失去角色特色時，介入引導。  
- **何時有用**：經過大量工具調用（Tool Call）或長篇大論後，AI 的角色設定（Persona）往往會淡化。本插件透過過濾機制，確保發送出去的訊息維持正確的表達式。

### 3. 🛡️ 區塊級代碼湮滅 (Code Annihilation)
- **功能描述**：全量攔截幻覺產生的 XML 標籤（如 `<response>`、`<function_call>`）及 JSON 片段。  
- **何時有用**：模型在思考過程中，意外將內部的工具指令或思考鏈（COT）當成聊天文字發出時。本功能採用「區塊級湮滅」技術，連皮帶骨切除所有代碼特徵，絕不漏掉任何一個括號（如 `}`）。

### 4. 🆘 嚴重損毀重寫機制 (Fallback Rewrite)
- **功能描述**：訊息損毀判定與兜底替換。  
- **何時有用**：如果一條回覆被代碼佔據了 90%，切除代碼後只剩 1、2 個殘渣字（如「} 好的」），插件會判定訊息已「損毀」，直接捨棄該回覆，並替換為管理員預設的「人格化道歉文案」。

### 5. ⏳ 真人感延遲分段發送 (Human-like Segmenting)
- **功能描述**：將長文按換行符切分，模擬真人輸入節奏。  
- **何時有用**：機器人一次性吐出 500 字會顯得極具「機器感」。本功能將訊息切分為多條獨立訊息，每條中間間隔自訂秒數（如 0.8s），營造出一句一句跟你說話的靈魂感。

### 6. 🔌 插件兼容性保障 (Transparent Compatibility)
- **功能描述**：系統指令的「透明穿透」技術。  
- **何時有用**：當你使用 `!reset` 或其他插件的功能時，本插件會自動判定內容長度與特徵，對於這類短小精悍的系統回覆，插件會「裝作不存在」直接放行。

## ⚙️ 配置項說明 (Manifest Spec)

| 配置項          | 類型   | 說明 | 預設值 |
|-----------------|--------|------|--------|
| rewrite_notice  | string | 當觸發修正（如代碼外洩）時，顯示給用戶的提示。 | ⚠️ [系統] 偵測到人格偏移... |
| fallback_text   | text   | 當訊息完全損毀（沒剩幾個字）時，重寫後的內容。 | （揉了揉眼睛）抱歉，我剛剛腦袋好像突然當機了... |
| delay_time      | number | 分段發送時，每段訊息之間的停頓秒數。 | 0.8 |

## 🚀 技術架構：如何確保「一勞永逸」？

- **物理斷路 (Physical Interception)**  
  不同於傳統插件只是修改文字，本插件在判定需要修正或分段時，會調用 `ctx.prevent_default()`，相當於物理剪斷了主程式原本要發出的訊號線，改由插件完全接管發送權。

- **證據化回報 (PowerShell Logging)**  
  安檢員的每一次「掃描」、「攔截」、「修正」都會帶上時間戳，詳細打印在 PowerShell 終端機。管理員能實時看到每一條訊息的安檢結果：  
  - ✅ 檢查通過，透明穿透：代表訊息正常，不介入。  
  - 🚨 發現代碼外洩！已執行區塊級物理湮滅：代表代碼被成功切除。  
  - 🆘 訊息已完全損毀！啟動兜底重寫機制：代表重寫功能生效。  

- **記憶體隔離 (Memory Safety)**  
  使用 `id(event)` 進行動態 ID 鎖定，防止在異步環境下產生無限迴圈發送，保證系統穩定性。
